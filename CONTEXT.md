# 프로젝트 컨텍스트 (CONTEXT)

## 1. 핵심 파일 구조
```text
GoldSafe/
 ├─ index.html      # 메인 단일 페이지
 ├─ style.css       # 전체 스타일 시트 (슬라이드쇼 애니메이션 포함)
 └─ images/         # 이미지 리소스 폴더
     ├─ sunil_1.png
     └─ sunil_2.png
 ├─ README.md           # 프로젝트 안내
 └─ CONTEXT.md          # 기술 명세 문서
```

## 2. 데이터 명세
이 프로젝트는 **정적(Static) 웹사이트**로, 별도의 데이터베이스(DB)나 백엔드 API를 사용하지 않습니다.
- 모든 정보(텍스트, 이미지 경로)는 `index.html` 내에 하드코딩되어 있습니다.
- 문의 접수는 웹사이트 내 DB 저장이 아닌, 사용자의 **전화 앱** 또는 **문자 앱**을 호출하는 방식(`tel:`, `sms:`)을 사용합니다.

## 3. 기술적 제약 사항
- **이미지 경로**: CSS 내 `background-image` 경로는 `images/파일명.png`로 상대 경로가 지정되어 있습니다. 따라서 `style.css`와 `images` 폴더의 상대적 위치가 `site` 폴더 구조를 유지해야 정상 작동합니다.
- **브라우저 호환성**: 최신 브라우저(Chrome, Edge, Safari, Firefox)에 최적화되어 있습니다. IE(Internet Explorer) 등 구형 브라우저에서는 슬라이드쇼 애니메이션이나 일부 레이아웃(Flexbox)이 의도대로 보이지 않을 수 있습니다.
- **자바스크립트 부재**: 현재 모바일 메뉴 토글(햄버거 버튼) 등의 동적 기능은 구현되어 있지 않으며, CSS Media Query를 통해 레이아웃만 변경되도록 처리되어 있습니다.

## 4. 개선 사항
- **모바일 메뉴**: 모바일 화면에서 네비게이션 메뉴가 너무 길어질 경우를 대비해, 접었다 펴는(Toggle) 기능을 JavaScript로 추가할 필요가 있습니다.
- **이미지 최적화**: 현재 원본 PNG 파일을 사용 중이나, 로딩 속도 개선을 위해 WebP 변환 및 이미지 용량 압축을 고려할 수 있습니다.
- **갤러리 기능**: 중고 금고 목록을 보여주기 위해, 추후 Grid 레이아웃을 활용한 갤러리 섹션을 보강할 수 있습니다.
